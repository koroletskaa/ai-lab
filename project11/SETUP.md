# Інструкція з налаштування та запуску

## Крок 1: Налаштування пристрою ESP32

1. Відкрийте Arduino IDE
2. Встановіть платформу ESP32 через менеджер плат
3. Встановіть бібліотеки:
   - U8g2 (для OLED)
   - ArduinoJson
4. Відкрийте `device/pulse_tracker.ino`
5. Змініть налаштування Wi-Fi:
   ```cpp
   const char* ssid = "ВАША_WIFI_МЕРЕЖА";
   const char* password = "ВАШ_ПАРОЛЬ";
   const char* serverURL = "http://IP_ВАШОГО_КОМП'ЮТЕРА:8000/api/pulse";
   ```
6. Завантажте код на ESP32

## Крок 2: Запуск Backend

1. Відкрийте термінал у папці `website/backend`
2. Створіть віртуальне середовище:
   ```bash
   python -m venv venv
   ```
3. Активація (Windows):
   ```bash
   venv\Scripts\activate
   ```
   (Linux/Mac: `source venv/bin/activate`)
4. Встановіть залежності:
   ```bash
   pip install -r requirements.txt
   ```
5. Запустіть сервер:
   ```bash
   python main.py
   ```

Сервер буде доступний на `http://localhost:8000`
Документація API: `http://localhost:8000/docs`

## Крок 3: Запуск Frontend

**Варіант А: Одночасний запуск обох серверів (рекомендовано)**

З кореня проєкту:
```bash
python start.py
```

Або на Windows:
```bash
start.bat
```

Або на Linux/Mac:
```bash
chmod +x start.sh
./start.sh
```

**Варіант Б: Окремі термінали**

1. Відкрийте термінал у папці `website/frontend`
2. Запустіть локальний сервер:
   ```bash
   python -m http.server 8080
   ```
3. Відкрийте браузер: `http://localhost:8080`

## Крок 4: Перевірка роботи

1. Переконайтеся, що ESP32 підключено до Wi-Fi
2. Переконайтеся, що пристрій відправляє дані на backend
3. У frontend перевірте статус підключення пристрою
4. Спробуйте додати запис емоції через форму

## Тестування API

Запустіть тестовий скрипт:
```bash
cd website/backend
python test_api.py
```

## Розв'язання проблем

### Backend не запускається
- Переконайтеся, що порт 8000 не зайнятий
- Перевірте встановлення всіх залежностей

### Пристрій не підключається
- Перевірте налаштування Wi-Fi
- Перевірте IP адресу сервера в коді пристрою
- Переконайтеся, що ESP32 і комп'ютер в одній мережі

### Frontend не відображає дані
- Перевірте, що backend працює
- Відкрийте консоль браузера (F12) для перевірки помилок
- Переконайтеся, що API_BASE_URL у `app.js` правильний
